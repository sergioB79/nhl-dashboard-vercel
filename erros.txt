1. O que está a acontecer no Action

Do log:

Failed to resolve 'statsapi.web.nhl.com' ([Errno -2] Name or service not known)


Isto quer dizer:

O runner do GitHub tenta ir a
https://statsapi.web.nhl.com/api/v1/teams
https://statsapi.web.nhl.com/api/v1/schedule?...

O domínio statsapi.web.nhl.com já não existe / não resolve DNS → NameResolutionError.

O teu código apanha a exceção, escreve warnings, mas no fim marca ok=True e grava games_cache.json (provavelmente quase vazio ou com dados antigos).

Ou seja:
✅ Action corre.
✅ Script não “rebenta”.
❌ Mas já não consegue buscar dados novos porque o endpoint foi desligado.

2. Root cause (não é culpa tua)

A NHL desligou a API antiga statsapi.web.nhl.com e passou tudo para a API nova:

Web API nova: https://api-web.nhle.com/v1/...

Stats API nova: https://api.nhle.com/stats/rest/en/...

Exemplo real: hoje em dia usam-se endpoints como:

https://api.nhle.com/stats/rest/en/team/powerplay?...   # devolve teamFullName, etc.


Portanto, o erro é exatamente isto:

O host statsapi.web.nhl.com deixou de existir → DNS falha → NameResolutionError.

Passo 3 — Decidir estratégia para a API nova

Soluçao –> Reescrever a sério para a nova API

Aqui a ideia é:

Substituir todos os endpoints statsapi.web.nhl.com por:

https://api-web.nhle.com/v1/schedule/{date} (calendário),

https://api-web.nhle.com/v1/gamecenter/{GAME_ID}/boxscore (períodos, golos, etc.),

https://api.nhle.com/stats/rest/en/team/... (registos, teamFullName, etc.).

Ajustar o parsing do JSON (os campos mudam de nomes).

Voltamos a ter:

nhl_periods_20242025.csv com golos por período por equipa/jogo;

games_cache.json completo com registos, period/clock corretos.

Isto é totalmente faisável, só é mais trabalho: é quase um mini-projeto novo.
